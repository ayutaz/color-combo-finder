# 4. 非機能要件

技術調査向けに大幅に簡略化。**「最低限これだけは守る」** という基準。

## 4.1 パフォーマンス

| ID | 要件 |
| --- | --- |
| NFR-01 | HEX 入力からチップ表示まで体感で遅延しない（数十 ms 程度） |

ピクセル単位の最適化やバンドルサイズ上限は **設けない**（必要になったら計測する）。

## 4.2 ブラウザ・実行環境

| ID | 要件 |
| --- | --- |
| NFR-10 | 開発者の常用ブラウザ（Chrome / Safari の最新版）で動作すること |
| NFR-11 | サーバー不要のクライアントサイド完結（静的ファイルのみ） |

レガシーブラウザ対応、オフライン動作、Service Worker は **不要**。

## 4.3 保守性・実装方針

| ID | 要件 |
| --- | --- |
| NFR-20 | 色計算ロジックは UI から独立した純関数として実装する |
| NFR-21 | TypeScript の strict モードを有効化する |
| NFR-22 | 色計算ロジックには単体テストを書く（カバレッジ数値目標は設けない） |

## 4.4 CI/CD

| ID | 要件 |
| --- | --- |
| NFR-30 | GitHub Actions で push / PR 時にテストを自動実行する |
| NFR-31 | main へのマージ後、自動で GitHub Pages にデプロイする |
| NFR-32 | テストが失敗した場合はデプロイされない |

## 4.5 意図的に対象外とする項目

技術調査が目的のため、以下は **明示的にスコープ外**：

- WCAG アクセシビリティ準拠
- レスポンシブデザイン
- ダーク／ライトモード
- 多言語対応
- ユーザーデータの永続化・プライバシー要件
- SEO
- パフォーマンス計測（Lighthouse 等）
