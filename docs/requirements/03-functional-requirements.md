# 3. 機能要件

技術調査用の試作のため、要件は **Must を最小限に絞り**、それ以外は Won't（初版では作らない）とする。

要件 ID は `FR-XX` 形式。

## 3.1 入力

| ID | 要件 | 区分 |
| --- | --- | --- |
| FR-01 | ベースカラーを HEX 文字列で入力できる（`#RRGGBB`） | Must |
| FR-02 | ブラウザ標準の `<input type="color">` でも入力できる | Must |
| FR-03 | 不正な HEX 入力は無視する（最低限のバリデーション） | Must |

`#RGB` 短縮形、RGB / HSL 直接入力、入力値のリアルタイム相互変換などは Won't（不要）。

## 3.2 パレット

| ID | 要件 | 区分 |
| --- | --- | --- |
| FR-10 | コード上にハードコードされたサンプルパレットを 1〜2 種類同梱する | Must |
| FR-11 | パレットの内容を画面上に確認できる（チップ表示） | Must |

パレットの動的な追加・編集・保存・命名・複数管理・インポート／エクスポートはすべて Won't。

## 3.3 組み合わせ計算

| ID | 要件 | 区分 |
| --- | --- | --- |
| FR-20 | 補色（complementary）を計算する | Must |
| FR-21 | 類似色（analogous, ±30°）を計算する | Must |
| FR-22 | トライアド（triadic, 120° 等分）を計算する | Must |
| FR-23 | モノクロマティック（明度違い）を計算する | Must |
| FR-24 | 計算結果はパレット内の最近傍色にマッピングする | Must |
| FR-25 | 「近さ」は CIEDE2000 で評価する | Must |

テトラード・スプリットコンプリメンタリー・スクエア・WCAG コントラスト比併記は Won't（必要になったら追加）。

## 3.4 表示

| ID | 要件 | 区分 |
| --- | --- | --- |
| FR-40 | 各組み合わせを横並びのチップで表示する | Must |
| FR-41 | チップに HEX 値を併記する | Must |

カラーホイール描画、UI プレビュー、ライト／ダーク切替、レスポンシブ、アニメーションは Won't。

## 3.5 デプロイ・自動化

| ID | 要件 | 区分 |
| --- | --- | --- |
| FR-50 | GitHub Pages に公開 URL を持つ | Must |
| FR-51 | main ブランチへの push をトリガーに GitHub Actions が自動でテストを実行する | Must |
| FR-52 | テスト成功時に GitHub Pages へ自動デプロイする | Must |
| FR-53 | プルリクエストでもテストが走り、結果が PR チェックに表示される | Must |

localStorage 永続化、URL ステート同期、お気に入り、エクスポート機能は Won't。
