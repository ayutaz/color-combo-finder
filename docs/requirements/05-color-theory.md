# 5. 色彩理論の前提

計算ロジックの根拠を最小限にまとめる。

## 5.1 使用する色空間

| 色空間 | 用途 |
| --- | --- |
| sRGB / HEX | 入出力、画面表示 |
| HSL | 色相環ベースの組み合わせ計算 |
| CIE Lab | ΔE による最近傍探索 |

変換は culori に任せる。自前実装はしない。

```
入力 HEX → (HSL で角度操作 | Lab で距離計算) → HEX 出力
```

## 5.2 色彩理論ルール

色相環（0°〜360°）上で、ベースカラーの色相 H に対して以下を計算する。

| ルール | 計算方法 | 出力色数 |
| --- | --- | --- |
| 補色 (Complementary) | H + 180° | 2 |
| 類似色 (Analogous) | H − 30°, H, H + 30° | 3 |
| トライアド (Triadic) | H, H + 120°, H + 240° | 3 |
| モノクロマティック (Monochromatic) | 同 H で L を等分 | 3〜4 |

スプリットコンプリメンタリー・テトラード・スクエア等は本フェーズでは対象外（[09-milestones §9.5](./09-milestones.md#95-後続テーマ着手未定) 参照）。

## 5.3 パレットへのマッピング

計算で得られた理想色 `C_target` は、パレット `P` の中で最も近い色 `C_match` に対応付ける：

```
C_match = argmin_{c ∈ P}  ΔE(c, C_target)
```

ΔE は **CIEDE2000** を採用する（culori の `differenceCiede2000` を使用）。

複数の `target` が同じ `C_match` にマップされることは許容する（パレット規模が小さいため）。
