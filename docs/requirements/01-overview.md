# 1. プロジェクト概要

## 1.1 目的

「事前に決まった数色のパレット」と「ユーザーが指定したベースカラー」を入力に、色彩理論に基づく組み合わせを計算して可視化する **技術調査用の試作ツール** を作る。

主眼は次の 3 点を実コードで確かめること：

1. 色相環ベースの組み合わせ計算（補色・類似色・トライアド等）
2. パレット内の最も近い色へのマッピング（色差 ΔE）
3. ブラウザでの軽量な可視化（チップ表示 + 必要なら色相環）

商用品質のツール開発ではないため、UI の作り込み・国際化・保存・共有機能などは **非目標** とする。

## 1.2 スコープ

### 含む（In Scope, Must）

- 単一ベースカラー入力（HEX 文字列）
- ハードコードされたサンプルパレット 1〜2 種（数色〜10 色程度）
- 主要な色彩理論ルール（補色・類似色・トライアド・モノクロマティック）の計算
- パレット内の最近傍色マッピング
- 結果のチップ表示

### 含まない（Out of Scope）

- 複数パレットの管理、保存、編集 UI
- パレットの JSON/CSS/SCSS 等へのエクスポート、画像書き出し
- URL ステート同期、お気に入り
- ダーク／ライトモード、レスポンシブの作り込み
- 多言語対応
- アクセシビリティの厳密対応（WCAG AA 達成は努力目標）
- カラーピッカー以外の入力 UI（RGB / HSL 直接入力など）

## 1.3 想定成果物

- ブラウザで開ける単一の SPA（または素の HTML + TS）
- **GitHub Pages にデプロイされた公開 URL**
- 色計算ロジックの単体テスト数本（アルゴリズム検証用）
- **GitHub Actions による CI/CD**（push → テスト → デプロイの自動化）

## 1.4 成功条件

- HEX を入力するとパレットから最も近い色を使った組み合わせがチップで表示される
- 色計算ロジックが UI から分離されていて、テストで挙動を確認できる
- main への push で GitHub Actions が自動でテストを走らせ、成功時に GitHub Pages にデプロイされる
